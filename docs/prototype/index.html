一、课程定位

以 DB 为体、AI 为用、协作为核心，立足人工智能时代，培养兼具数据库系统理论与实践应用能力的高素质人才。

AI 融合NL2SQLOBE⌛ 思政预留位

二、课程目标

课程目标 1（知识目标）
掌握数据库基本概念、数据模型、关系模型及关系代数等知识。

课程目标 2（能力目标）
掌握 SQL 语言的 DDL、DML 使用，能独立完成复杂数据查询与视图操作。

课程目标 3（素养目标）
掌握嵌入式 SQL、游标及存储过程的使用；结合 AI 工具辅助数据库设计。

三、教学内容与学时分配

第一章 · 数据库概述（4 学时）

  • 数据库系统的应用价值
  • 数据库的定义与基本概念
  • 数据库系统的标准体系结构
  • 数据库系统在 AI 时代的生态

思政预留：⌛ 思政案例库接入后自动填充

第二章 · 关系模型（6 学时）

  • 关系数据库的逻辑结构
  • 关系的定义、性质与基本操作
  • 关系完整性约束
  • 关系代数的五大基本操作
π name, age (σ age≥18 ∧ dept='IS' (Student))

第三章 · SQL 语言（10 学时）

  • SQL 语言的功能特点与体系结构
  • 数据定义（DDL）
  • SQL 简单查询 / 连接查询
  • 数据更新（DML）
  • NL2SQL 人机对比
  • 实验 1：SQL 查询
SELECT s.name, c.title, sc.grade FROM Student s JOIN SC sc ON s.sid = sc.sid JOIN Course c ON sc.cid = c.cid WHERE sc.grade >= 85;

RULE 01

学时自适应分档

根据总学时自动调整章节数与每章字数。不同学时档位对应不同的课程密度。

if hours >= 48:  units=6-8, words=2000-3000
elif hours >= 32: units=4-6, words=1500-2000
else:             units=3-4, words=1000-1500
