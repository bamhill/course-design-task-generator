# 项目版本记录

> 项目：课程设计任务书智能生成（信管）  
> 聚焦：固定大纲 + 教师个性化需求 → 两步出任务书（初版→优化→定稿）  
> 文件：`index.html` — 单文件 HTML 设计原型

## v0.1 — 2026-06-17（首版 · Editorial Tech）

**特征：** 暗色沉墨底 + 学术金 + 数据青 + AI 紫四色配色  
**字体：** Fraunces 衬线大标题 + Geist 无衬线正文 + JetBrains Mono 代码  
**项目名：** CurriculumForge（课程设计文档生成器）

### 页面结构
- **Hero 区：** 大标题 + 副标题 + 右侧 JSON 终端卡片（展示 course_info.json）
- **核心能力：** 3 列：Data-Driven / Rule Engine / Output
- **示范课程：** 数据库原理与应用 6 单元完整展示（课程序号 04042603a）
- **工作流：** 四步流程（A→B→C→D）
- **生成器：** 多 tab 表单 + 右侧实时预览（含 SQL 代码块）
- **规则引擎：** 3 条规则伪代码（学时分档 + 学情层次 + 思政注入 RULE 03）
- **导出区：** .md / .docx / .json / .pdf

### 思政预留
- RULE 03 展示「思政案例自动注入」+ Planned 状态

**触发：** 用户要求用 frontend-design 设计课程设计文档生成器

## v0.3 — 2026-06-17（方案精简转向）

**修改原因：** 用户确认核心需求是「两步流程」——固定大纲+个性需求→初版任务书→教师优化→优化报告+对比表+定稿，不需要大平台。

### 改动清单

#### 🎯 方向转换
- 旧 v0.2：通用课程设计文档生成器（大平台、6 区页面）
- 新版：**两步精简流程**，聚焦三份输入 + 三份输出

#### 🏠 Hero 区重构
- 标题：课程设计文档生成 → **课程设计任务书智能生成**
- 示范课程：数据库原理与应用（通用）→ **数据库原理与应用（SQL-Server）实践**
- 课程序号：04042603a → **04040313b**（课设专用编号）
- 学分学时：3 学分 48 学时 → **1 学分 1 周（16 学时）**
- 统计栏：数据源/规则/输出格式 → **输入文档/生成步骤/产出文件/思政**

#### 📋 工作流区重构
- 旧：四步流程（A→B→C→D）
- 新：**两步流程**（① 录入个性需求→初版任务书 → ② 录入优化需求→报告+对比表+定稿）
- 中间用箭头连接，一目了然

#### 📄 方案文档
- 创建 `docs/课程设计任务书智能生成方案.md`

**触发：** 用户与老师沟通后确认需求是两步精简流程
