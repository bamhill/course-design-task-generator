# 会话交接规则

## 强制日期标签
所有 memory/ 条目必须带 `[YYYY-MM-DD]` 前缀。没有日期 = bug。

## 用户提醒优先
用户主动要求记录的内容，无条件写入。

## Session 结束时 Claude 必须执行

> 本清单与根 `.claude/rules/session-workflow.md` 联动，根规则为权威来源。

### 1. 记录本子项目
- [ ] `docs/开发类/会话经验教训.md` — 日期标题 + 具体教训
- [ ] `docs/开发类/方案探索过程.md` — 如有头脑风暴（日期+淘汰方案+决定理由）
- [ ] `docs/开发类/方案架构设计过程.md` — 如有架构变更（日期+变更+原因）
- [ ] `docs/开发类/需求变更记录.md` — 如有需求变更（日期+变更+原因+影响范围+决策）
- [ ] `memory/已验证经验.md` — 如有经实战验证的方法（`[YYYY-MM-DD]` 前缀，新建时更新 MEMORY.md 索引）
- [ ] `memory/踩坑记录.md` — 如有非平凡 bug 解决（同上）
- [ ] `memory/通用技巧.md` — 如有可跨项目复用的操作（同上）
- [ ] `memory/跨项目设计参考.md` — 如发现兄弟项目可借鉴特性（同上）
- [ ] `tasks/待办.md` + `tasks/已完成.md` — 更新任务状态

### 2. 晋升审计
读取最近 7 天 memory/ 条目，找**同一模式在 3+ 不同日期出现**：
- [ ] 提出 2-4 个晋升候选
- [ ] 口头向用户确认，确认后写入
- [ ] 晋升后清理重复条目（热缓存 ≤200 行）

### 3. 回流总根
- [ ] 通用 → 写入 `../../memory/` 对应文件（带日期标签）
- [ ] 项目专属 → 不回流

### 4. 项目总结（阶段结束时执行）
- [ ] 对比初始需求文档与最终交付，写 `docs/开发类/需求变更记录.md`
- [ ] 更新需求文档版本号，反映最终系统
- [ ] 写 `docs/开发类/方案探索过程.md`（记录关键设计讨论和范式选择）
- [ ] 运行 `python ../../promotion_scanner.py` 生成晋升审计报告
- [ ] 更新 `../../docs/项目进度.md` 中本项目的状态
- [ ] 写入 `../../logs/完整会话记录_YYYYMMDD_{主题}.md`（根级 logs/）
- [ ] 按需生成 out_ 模板（见 `../../00_模板/out_模板/out_模板规范.md`）
