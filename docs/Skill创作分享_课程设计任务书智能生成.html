1、Skill 简介

课程设计任务智能分配系统 是一个让同一门课的多位老师都能个性化生成自己的课程设计任务书的 Claude Code Skill + Web 工具组合。核心逻辑：同一个固定大纲，不同老师填不同需求，生成属于自己班的任务书。

你只需要在网页上填写自己班的班级人数、教学倾向和任务分配策略，系统就会自动生成：

  • 每位学生的个性化课程设计任务书（WORD 版）
  • 教师分配总表（谁做了什么任务一目了然）
  • 优化总结报告（修改了什么、为什么改、对比表）

适合谁用：高校同一门课由多位老师平行授课的场景——每位老师都有自己的教学风格和侧重点，需要在自己的班级里自由发挥，但大纲是统一的。

2、使用场景

为什么想做它？

一位老师的真实日常：

  • 同一门课，多位老师教：计算机网络课设，信管专业 3 位老师各带不同班级
  • 每位老师的风格不同：张老师侧重动手、李老师侧重理论、王老师项目导向
  • 学生水平也不一样：有的班 35 人，有的班 70 人，有的需要分层
  • 但大纲是同一份：不能用同一份任务书应付所有班
  • 今年教完明年还教：同样的课、同样的老师，但学生换了、时间变了
  • 去年的任务书不能直接用：改年份、调题目、换班级，几乎重写

刚需：多位老师共用同一份大纲，各自生成自己班的个性化任务书。

之前遇到的麻烦

痛点具体表现
任务分配难70 人不能做一样的题目，但出 10+ 个不同题目太累
任务书撰写繁每个题目都要写任务书，七八份写到手软
格式统一难不同题目的任务书风格各不同，学生看糊涂
思政融入硬要求体现课程思政，但每道题怎么自然融入
不区分学生差异优等生和基础弱的用同一份任务
优化无记录改了几版没有文档记录，教务检查拿不出依据

做出来之后省掉了什么

之前现在
手动出 10+ 题目 + 写 70 份任务书：2-3 天Web 填参数，自动分配，一键出所有文件
思政硬写：每题 30 分钟自动按阶段融入，正反例对照
优化靠微信记录自动输出优化总结报告 + 对比表
不区分学生差异支持分层：基础题 / 标准题 / 挑战题
下一年重新来改一改参数，复用全部模板

3、创作过程

核心思路

整个系统的架构分 3 层：

数据层 → 分配引擎 → 输出层
固定大纲 + 教师需求 + 思政参考 → 任务分配规则 + 分层策略 → WORD 任务书 + 分配总表 + 优化报告

关键创新：不是"生成一份任务书"，而是"为 N 个学生/小组分别分配不同的设计任务"。

Step 1：选择数据源

最终选择 固定大纲 + Web 网页 + 对话式：

  • 理论大纲和课设大纲作为骨架
  • 教师通过 Web 网页填写需求（不上手代码）
  • 思政参考文档提供写作模式

网页设计：教师先用 Web 完成任务分配

入口是一个暖白学术风的 Web 页面（web_app/），教师通过它完成任务分配 + 需求录入 + 文件下载：

  • 选择课程：计算机网络 / 数据库原理与应用
  • 填写班级信息：专业、班级、人数
  • 选择任务分配策略：全部相同 / 不同题目 / 分层分配（基础/标准/挑战）
  • 设置题目数量和分组方式
  • 上传补充材料（可选）：支持文件拖拽上传
  • 一键生成：后端自动分配 → 打包下载所有 WORD 文件

Step 2：设计思政融入

目标：自然融入、不硬贴标签、正反例对照。

阶段思政点正例反例
需求分析数据安全意识引导讨论数据保密直接给真实企业数据
概念设计系统思维E-R 图展示全局数据流转只做单表设计
逻辑设计工程严谨规范化分析不能跳直接给完整表结构
数据库实现工匠精神要求代码注释和规范命名运行就行
功能调试数据完整性底线去掉约束演示后果直接告诉答案
团队协作责任意识答辩追问模块间影响只收个人报告

Step 3：两步流程

第一步：教师给需求 + 选分配策略 → 自动生成学生任务书 + 分配总表
                        ↓
第二步：教师审阅后提优化需求 → 输出优化报告 + 对比表 + 定稿

Step 4：生成 WORD

用 python-docx 生成带格式的 WORD 文档，宋体 11pt、表格居中、思政提示蓝字标注，可直接提交教务处。

4、使用步骤

推荐 对话式（零门槛，推荐）

教师直接在 Claude Code 中说：

"@course-design-task-generator 帮我分配计算机网络课程设计任务，信管 2024 级 70 人，分 10 个不同题目"
// 或者更简单的：
"帮我生成计网课设任务书，信管 2024 级 70 人"
"生成数据库原理与应用的课设任务书，信管专业 42 人"
"第二步优化：概念设计阶段拆成两天，平时考核调成 30%"

网页版（适合不熟悉命令行的老师）

部署到服务器后：
打开网页 → 选课程 → 填班级/人数 → 选分配策略 → 一键生成 → 下载所有 WORD

本地运行：

pip install fastapi uvicorn python-multipart python-docx
cd web_app
uvicorn main:app --host 0.0.0.0 --port 8000

命令行直接运行

git clone https://github.com/bamhill/course-design-task-generator.git
pip install python-docx
python scripts/generate_cntask_doc.py
