## 1、Skill 简介

**课程设计任务智能分配系统** 是一个让同一门课的**多位老师都能个性化生成自己的课程设计任务书**的 Claude Code Skill + Web 工具组合。

核心逻辑：**同一个固定大纲，不同老师填不同需求，生成属于自己班的任务书。**

你只需要在网页上填写自己班的班级人数、教学倾向和任务分配策略，系统就会自动生成：

- **本班每位学生的个性化课程设计任务书**（WORD 版）
- **教师分配总表**（谁做了什么任务一目了然）
- **优化总结报告**（修改了什么、为什么改、对比表）

**适合谁用**：高校同一门课由多位老师平行授课的场景——每位老师都有自己的教学风格和侧重点，需要在自己的班级里自由发挥，但大纲是统一的。

## 2、使用场景

### 为什么想做它？

一位老师的真实日常：

- **同一门课，多位老师教**：计算机网络课设，信管专业 3 位老师各带不同班级
- **每位老师的风格不同**：张老师侧重动手实践、李老师侧重理论深度、王老师要求项目导向
- **学生水平也不一样**：有的班 35 人，有的班 70 人，有的需要分层（基础/标准/挑战）
- **但大纲是同一份**：不能用同一份任务书应付所有班

还有另一层刚需——**同一门课跨学期复用**：

- **今年教完明年还教**：同样的课、同样的老师，但学生换了、时间变了
- **去年的任务书不能直接用**：改年份、调题目、换班级，几乎重写
- **积累了 3 年的材料没有沉淀**：每年从零开始

这就是刚需：**多位老师共用同一份大纲，各自生成自己班的个性化任务书。**

### 之前遇到的麻烦

| 痛点 | 具体表现 |
|------|---------|
| **任务分配难** | 70 人不能做一样的题目，但出 10+ 个不同题目太累 |
| **任务书撰写繁** | 每个题目都要写任务书，七八份写到手软 |
| **格式统一难** | 不同题目的任务书风格各不同，学生看糊涂 |
| **思政融入硬** | 要求体现课程思政，但不知道每道题怎么自然融入 |
| **不区分学生差异** | 优等生和基础弱的用同一份任务，两极都不满足 |
| **优化无记录** | 改了几版没有文档记录，教务检查拿不出依据 |

### 做出来之后省掉了什么

| 之前 | 现在 |
|------|------|
| 手动出 10 个题目 + 写 70 份任务书：2-3 天 | Web 网页填参数，自动分配，一键出所有文件 |
| 思政硬写：每题 30 分钟 | 自动按阶段融入，正反例对照 |
| 优化靠微信记录 | 自动输出优化总结报告 + 对比表 |
| 不区分学生差异 | 支持分层：基础题 / 标准题 / 挑战题 |
| 下一年重新来 | 改一改参数，复用全部模板 |

**一句话总结**：从"手动出题 2-3 天 + 手动写任务书 1 人 1 份"到"网页填几个参数 → 自动分配任务 → 一键下载所有 WORD 任务书 + 分配总表 + 优化报告"。

## 3、创作过程

### 核心思路

整个系统的架构分 3 层：

```
数据层 → 分配引擎 → 输出层
固定大纲 + 教师需求 + 思政参考 → 任务分配规则 + 分层策略 → WORD 任务书 + 分配总表 + 优化报告
```

**关键创新**：不是"生成一份任务书"，而是"为 N 个学生/小组分别分配不同的设计任务"。


对比了几个输入方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| 纯对话式 | 零门槛 | 没有固定大纲，生成质量不稳 |
| 固定大纲 + 对话 | 生成质量高，大纲不可改 | 需要提前准备好大纲文件 |
| 全自动抓取 | 不需要教师动手 | 抓取的内容不准，不如大纲权威 |

最终选择 **固定大纲 + Web 网页 + 对话式**：
- 理论大纲和课设大纲作为骨架
- 教师通过 Web 网页填写需求（不上手代码）
- 思政参考文档提供写作模式

### 网页设计：教师先用 Web 完成任务分配

入口是一个暖白学术风的 Web 页面（`web_app/`），教师通过它完成**任务分配 + 需求录入 + 文件下载**：

- **选择课程**：计算机网络 / 数据库原理与应用
- **填写班级信息**：专业、班级、人数
- **选择任务分配策略**：全部相同 / 不同题目 / 分层分配（基础/标准/挑战）
- **设置题目数量和分组方式**
- **上传补充材料（可选）**：支持文件拖拽上传
- **一键生成**：后端自动分配 → 打包下载所有 WORD 文件

教师端不需要写任何代码，打开浏览器就能用。

### Step 2：设计任务书结构

参考了学校课设大纲的标准结构：

```
一、课程设计目标（从大纲课程目标提炼）
二、设计任务（2 大任务 × 4-5 阶段）
三、进度安排（天数表）
四、思政融合要求（每阶段 1 点）
五、考核标准（权重 + 考查点）
六、交付物清单
七、参考资料
```

### Step 3：思政融入设计

这是最花心思的部分。目标是：**自然融入、不硬贴标签、正反例对照**。

参考了《管理信息系统》思政文档的写作风格：

| 阶段 | 思政点 | 正例 | 反例 |
|------|--------|------|------|
| 需求分析 | 数据安全意识 | 引导讨论数据保密的重要性 | 直接给真实企业数据练习 |
| 概念设计 | 系统思维 | E-R 图展示全局数据流转 | 只做单表设计，不看全局 |
| 逻辑设计 | 工程严谨 | 规范化分析不能跳 | 直接给完整表结构 |
| 数据库实现 | 工匠精神 | 要求代码注释和规范命名 | 运行就行，不要求注释 |
| 功能调试 | 数据完整性底线 | 去掉约束演示数据不一致后果 | 直接告诉答案 |
| 团队协作 | 责任意识 | 答辩追问模块间影响 | 只收个人报告 |

### Step 4：两步流程设计

这是 Skill 的核心流程，从教师视角设计：

```
第一步：教师给需求 → 自动生成任务书（初版）
                    ↓
第二步：教师审阅后提优化需求 → 输出 3 件套
    ① 优化总结报告（简述改了什么）
    ② 改动对比表（原内容 / 改后 / 改因）
    ③ 任务书（定稿）
```

### Step 5：生成 WORD 文档

用 `python-docx` 生成带格式的 WORD 文档：

- 宋体正文 11pt，标题黑体
- 1.5 倍行距
- 表格上下左右居中
- 思政提示用蓝字标注
- 生成的 .docx 可直接提交教务处

关键代码（精简版）：

```python
from docx import Document
doc = Document()
# ... 逐章生成内容
doc.save('课程设计任务书_初版.docx')
```


## 5、效果展示

### 输入

```
"帮我生成计算机网络课程设计任务书，信管 2024 级 70 人"
```

### 输出

**生成的 WORD 文档包含**：

- **封面信息**：课程名称、编号、学分、适用专业
- **课程设计目标**：3 个目标（设计能力、配置能力、应用与安全）
- **设计任务**：2 大任务 × 4 阶段（含思政融入提示）
- **进度安排**：5 天详细表

- **考核标准**：平时 40% + 答辩 20% + 报告 40%
- **交付物清单**：6 项交付物
- **参考资料**：教材 + AI 工具（Packet Tracer AI / Wireshark AI / 通义灵码）

**视觉格式**：

| 元素 | 效果 |
|------|------|
| 正文 | 宋体 11pt，1.5 倍行距 |
| 标题 | 黑体加粗，层级分明 |
| 表格 | 上下左右居中，边框完整 |
| 思政提示 | 蓝字标注，融入在教学要求中 |
| 输出格式 | .docx，可直接提交教务处 |

**第二步优化输出**：

| 产出 | 说明 |
|------|------|
| 优化总结报告 | 简述改了什么、为什么改 |
| 改动对比表 | 原内容 / 改后 / 改因 |
| 任务书（定稿） | 融入优化后的最终版 |

## 7、总结与思考

### 效率提升

| 指标 | 之前 | 现在 | 提升 |
|------|------|------|------|
| 生成一份任务书 | 2 小时手写+排版 | 10 秒出 WORD | **720 倍** |
| 融入思政 | 1 小时硬写 | 自动生成，正反例对照 | **免人工** |
| 优化报告 | 靠聊天记录 | 自动输出对比表 | **有据可查** |
| 每学期多门课 | 逐个手动 | 改一下专业名 | **10 秒换课** |

### 创作心得

1. **固定大纲是最佳数据源**：一开始想过全自动生成，但发现没有大纲兜底的内容质量不可控。固定大纲 + 教师对话的组合效果最好。

2. **思政要"润物细无声"**：不单独设思政章节，融入在每个阶段的教学要求中。正反例对照让教师一看就懂什么叫"融入了"、什么叫"过了"。

3. **两步流程是关键创新**：初版 + 优化报告的分离，让老师既有"可用版本"快速下发，又有"优化记录"给教务检查，两个需求都满足了。

4. **版本记录比想象重要**：每次改动都存档为一个独立文件（`v0.1→v0.9`），方便回溯。这个习惯是从这个项目学到的。
